Java8 Features - BiPredicate Functional Interface

In this tutorial, We are going to learn about BiPredicate Functional Interface which was introduced as part of Java 8. As learned in previous tutorial about Functional Interface, we have only one abstract method in Predicate Function Interface called test().As name says this function evaluate an operation to return boolean result and it takes two arguments.

Methods in Predicate

MethodName type Description
test(T t,U u) abstract Evaluate the predicate and return boolean value
and(BiPredicate<? super T,? super U> other) default Chain multiple predicate function and apply LOGICAL AND condition. Checks if both predicate function return true.
or(BiPredicate<? super T,? super U> other) default Chain multiple predicate function and apply LOGICAL OR condition. Checks if atleast one predicate function return true.
negate() default Negate the actual response evaluate by predicate.

Example and explanation

  • test(T t,U u)

T AND U are the inputs given to the bipredicate function. In the below example , we are checking which is the greatest number. 
BiPredicate findGreaterNumber = (a, b) -> a > b;
findGreaterNumber.test(10, 6);

  • and(BiPredicate<? super T,? super U> other)

This method takes another predicate as input and evaluate both the predicate and check whether both returns true. we need to call test() to pass inputs to this function. This is a default method in Predicate Interface,so we are not breaking the rule of Function Interface(one abstract method) 
BiPredicate findGreaterNumber = (a, b) -> a > b;
BiPredicate notZeroCheck = (a, b) -> (a > 0 && b > 0);
 notZeroCheck.and(findGreaterNumber).test(5, 0);

  • or(BiPredicate<? super T,? super U> other)

This method takes another predicate as input and evaluate either one of the predicate returns true. we need to call test() to pass input to this function. 
BiPredicate findGreaterNumber = (a, b) -> a > b;
BiPredicate notZeroCheck = (a, b) -> (a > 0 && b > 0);
notZeroCheck.or(findGreaterNumber).test(5, 0);

  • negate()

This method negate the value returned by the test(). 
BiPredicate findGreaterNumber = (a, b) -> a > b;
BiPredicate notZeroCheck = (a, b) -> (a > 0 && b > 0);
notZeroCheck.or(findGreaterNumber).negate().test(5, 0);

Complete Code For Predicate

	
package com.weblearner.functionallambda;

import java.util.function.BiPredicate;

public class BiPredicateFunctionalInterface {

	public static void main(String[] args) {

		BiPredicate findGreaterNumber = (a, b) -> a > b;

		System.out.println("Find greater number  -->" + findGreaterNumber.test(10, 6));

		BiPredicate notZeroCheck = (a, b) -> (a > 0 && b > 0);

		System.out.println("BiPredicate And Condition ->" + notZeroCheck.and(findGreaterNumber).test(5, 0));

		System.out.println("BiPredicate OR Condition ->" + notZeroCheck.or(findGreaterNumber).test(5, 0));

		System.out.println("BiPredicate OR Condition ->" + notZeroCheck.or(findGreaterNumber).negate().test(5, 0));
	}

}
