Kubernetes(k8s) Namespace

Overview

Namespaces provide a scope for names. Names of resources need to be unique within a namespace, but not across namespaces. Namespaces cannot be nested inside one another and each Kubernetes resource can only be in one namespace. Namespaces are a way to divide cluster resources between multiple users

About Demo

In this tutorial, we are going to learn about creating namespace and pod inside a namespace.

Prerequisite

  • Oracle Virtual Box - Installation Link
  • Minikube & Kubectl- Installation Link

pod-definition-namespace.yaml

To create a POD inside any specific namespace. We need to specify the namespace tag in the metadata section of the pod definition file. 
apiVersion: v1
kind: Pod
metadata:
 name: namespace-pod
 namespace: mynamespace
 labels:
  app: namespace-pod
  type: service
spec:
 containers:
  - name: service-pod
    image: nginx

Start Minikube

Open command prompt and navigate to the location where we have minikube.exe. Run the below command to start the minikube in virtual machine.
  • minikube start --vm-driver=virtualbox

Namespace Creation

Creating a namespace is a simple command. Please see below
  • kubectl create namespace mynamespace

Pod in Namespace

We have saved the above pod yaml as "pod-definition-namespace.yaml" and we will use create command as below to create pod in mynamespace.
  • kubectl create -f pod-definition-namespace.yaml
