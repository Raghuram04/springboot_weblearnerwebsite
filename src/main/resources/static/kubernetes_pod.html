Kubernetes(k8s) Pod Creation and Basic Commands

Overview

Kubernetes is a portable, extensible, open-source platform for managing containerized workloads and services, that facilitates both declarative configuration and automation. It has a large, rapidly growing ecosystem. Kubernetes services, support, and tools are widely available.

About Demo

In this tutorial, we are going to learn about kubernetes using Minikube. We will start with basic POD creation and useful commands to get pod and node details.

Prerequisite

  • Oracle Virtual Box - Installation Link
  • Minikube & Kubectl- Installation Link

pod-definiton.yaml

To create a pod, we will write a simple yaml file with pod definition and create this pod in minikube. Most of the kubernetes definition file will have the below object.
Element Description
apiVersion Version of the Kubernetes API used to create the object. Version varies for objects. Please check this link
kind Kind of object we are trying to create like POD,REPLICASET,DEPLOYMENT,SERVICE etc.
metadata Data that help us to identify the particular object.
spec Describe the actual configuration needed for the object.

We are creating a Pod, so we have defined apiVersion as v1,kind as Pod,metadata to identify the pod. We have defined labels. We will see in upcoming tutorial how to use label to identify the pod. spec defines the container definition of the pod. In our example we have used "nginx" image. 
apiVersion: v1
kind: Pod
metadata:
 name: pod-example
 labels:
  app: sample-pod
  type: service
spec:
 containers:
  - name: nginx-pod
    image: nginx

Start Minikube

Open command prompt and navigate to the location where we have minikube.exe. Run the below command to start the minikube in virtual machine.
  • minikube start --vm-driver=virtualbox

Pod Creation

We will create a new pod using the pod definition file given above. We have named it as pod-definition.yaml and we will use create command to create new pod. Please see the command below
  • kubectl create -f pod-definition.yaml
  • kubectl get pods
  • kubectl describe pod pod-example
  • kubectl logs pod-example

Node Details

All the pods will be created inside minikube node. We can validate the node details and also check which node pod is running. Use the below commands.
  • kubectl get nodes
  • kubectl describe node minikube
  • kubectl get pods -o wide
