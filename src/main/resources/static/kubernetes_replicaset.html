Kubernetes(k8s) ReplicaSet

Overview

ReplicaSet defines the desired number of Pod which should be active in the given time. It helps us to attain High availability of service. In case if a pod goes down, new pod will be created. It will always maintains the minimum number of pod as specified in the definition file.

About Demo

In this tutorial, we are going to learn about ReplicaSet and configuring the pod scalability.

replicaset-definition.yaml

Below is the replicaset definition file. spec template defines the behavior of the replicaset. Selector will get the pods which need to be grouped under this replicaset. In this example , our pod label is "sampleapp". Pods created with that label will be considered for replication even if not defined within the same replicaset definition file.
apiVersion: apps/v1
kind: ReplicaSet
metadata:
 name: myapp-replica
 labels:
   app: myapp-replica
   type: back-end
spec:
 template:
  metadata:
   name: myapp-pod
   labels:
    app: myapp
    type: sampleapp
  spec:
   containers:
    - name: nginx-containers
      image: nginx
 replicas: 3	  
 selector:
  matchLabels:
   type: sampleapp

Start Minikube

Open command prompt and navigate to the location where we have minikube.exe. Run the below command to start the minikube in virtual machine.
  • minikube start --vm-driver=virtualbox

ReplicaSet Creation

We have saved the above definition file as "replicasets-definition.yaml". We will use create command to generate replicaset object.
  • kubectl create -f replicasets-definition.yaml
