Kubernetes(k8s) Secret

Overview

Secret are used to store and manage sensitive information.Pod can use this secret as environment variable or as a file mounted as volume. There are multiple secret type in kubernetes,we are going to see about Opaque type.

About Demo

In this tutorial, we are going to learn about Secret and different ways of creating secret in kubernetes.

Prerequisite

  • Oracle Virtual Box - Installation Link
  • Minikube & Kubectl- Installation Link

Start Minikube

Open command prompt and navigate to the location where we have minikube.exe. Run the below command to start the minikube in virtual machine.
  • minikube start --vm-driver=virtualbox

secret.yaml

There are multiple ways of creating secret in kubernetes. yaml definition file is one way to create a secret and reuseable.
apiVersion: v1
kind: Secret
metadata: 
 name: db-secret
data:
 PASSWORD: cGFzcw==

Secret Creation

We will create a new secret using the definition file given above. We have named it as secret-definition.yaml and we will use create command to create new pod. Please see the command below. Also we can create directly using --from-literal and --from-file
  • kubectl create -f secret-definition.yaml
  • kubectl create secret generic mysecret --from-literal=mypass="pass123"
  • kubectl create secret generic mysecretfile --from-file=secret_file.txt

Secret in POD

We are creating a pod definition file with secret. We are using the db-secret object and passing it as environment variable. 
apiVersion: v1
kind: Pod
metadata: 
  name: webapp1
spec: 
  containers: 
    - 
      envFrom:
        - 
          secretRef: 
            name: db-secret
      image: nginx
      name: mywebapp
