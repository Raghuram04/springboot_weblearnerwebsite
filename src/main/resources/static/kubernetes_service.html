Kubernetes(k8s) Service

Overview

Service is an abstraction which defines a logical set of Pods and a policy by which to access the pod.Generally we will specify the pod using selector.

About Demo

In this tutorial, we are going to learn about services in kubernetes.We will show demo for ClusterIP and NodePort Service in kubernetes.

Prerequisite

  • Oracle Virtual Box - Installation Link
  • Minikube & Kubectl- Installation Link

service-definition-clusterIP.yaml

This is default service type in kubernetes used to communicate between the pods in the cluster.In this example, we have defined selector as sampleapp.So all the pods with type as sampleapp will be part of the cluster service.
apiVersion: v1
kind: Service
metadata:
 name: myapp-clusterservice
spec:
 type: ClusterIP    ## this is default type. we can remove this
 ports:
  - targetPort: 80    
    port: 80
 selector:
  app: myapp
  type: sampleapp

Start Minikube

Open command prompt and navigate to the location where we have minikube.exe. Run the below command to start the minikube in virtual machine.
  • minikube start --vm-driver=virtualbox

NodePort Service

We have saved the above definition file as "service-definition-clusterIP.yaml". We will use create command to generate replicaset object.
  • kubectl create -f service-definition-clusterIP.yaml

service-definition-nodeport.yaml

Exposes the Service on each Node's IP at a static port.Service, from outside the cluster, by requesting NodeIP:NodePort 
apiVersion: v1
kind: Service
metadata:
 name: weblearner-service
spec:
 type: NodePort
 ports:
  - targetPort: 8085
    nodePort: 30018
    port: 8085
 selector:
  app: weblearner
  type: rest_service
