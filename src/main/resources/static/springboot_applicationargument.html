Spring Boot - Accessing Application Argument

Overview

Spring boot provides the feature to get the application argument used during start up in the code.Command line argument will overwrite the application.properties argument values

About Demo

In this Demo, we will show the implementation of accessing the command line argument in spring class using ApplicationArguments

Prerequisites

  • Maven 3.3.9
  • Java 8
  • Eclipse

pom.xml

Below dependencies are enough to develop the Spring MVC application using Spring Boot . Link to source code is available at the bottom of the page

SpringBoot Class

We will access the application arguments using @Value annotation in the code. SpringbootApplicationargsApplication implements ApplicationRunner and overrides the run(ApplicationArguments args) which will be executed in the start up. 3 parameters(name,location,port) are passed as the argument in the startup. server.port is defined in the application.properties with 8082. We will overwrite the value with command line argument 
				
@SpringBootApplication
public class SpringbootApplicationargsApplication implements ApplicationRunner {
	
	@Value("${name}")
	private String name;
	
	@Value("${location}")
	private String location;
	
	@Value("${server.port}")
	private String port;
	
	

	public static void main(String[] args) {
		SpringApplication.run(SpringbootApplicationargsApplication.class, args);
	}

	@Override
	public void run(ApplicationArguments args) throws Exception {
		
		System.out.println("Name -->"+ name);
		System.out.println("location -->"+ location);
		System.out.println("port -->"+ port);

	}

}

application.properties

Including server.port in the application file. This value will be overrided.

Result:


Please see the log during start up. server.port value is replaced as 8085 instead of 8082.Run below command while building
$ mvn spring-boot:run -Dspring-boot.run.arguments="--name=Test --location=US --server.port=8085"
